Setup Guide

Set up your AI worker without leaking secrets

This guide groups the environment variables by the order most people need them: deployment settings, required core values, generated secrets, public domain values, then optional integrations. Use blank placeholders here, and paste real secrets only into Render environment variables.

Start Here: what this setup does

  • Creates a deployable backend your AI assistant can use as a tool workspace.
  • Uses MCP as the protocol layer for assistant-to-tool communication.
  • Supports OAuth, token signing, optional bearer-token testing, memory, and grouped business integrations.
  • Keeps real secrets out of the public website and out of GitHub.

Safety rule

Do not paste real API keys, OAuth refresh tokens, client secrets, signing secrets, or bearer tokens into this website or any public repo file. The blocks below intentionally use blank values or placeholders. Real values belong in Render environment variables only.

Generate your secrets

Use these buttons to generate local placeholder secrets for your Render environment variables. They are not sent anywhere by this website. After copying them into Render, keep them private.

MCP_OAUTH_CLIENT_SECRET
Used by the static MCP OAuth client.
MCP_SERVER_BEARER_TOKEN
Optional simple token path for scripts and manual testing.
MCP_TOKEN_SIGNING_SECRET
Used to sign MCP access and refresh tokens.
OAUTH_STATE_SECRET
Used to protect OAuth state handling.

Public domain values

After Render gives you a domain, replace the placeholders with your actual backend service URL. Keep the scheme consistent, usually https://.

Google and GitHub integrations

Use these when you want the AI worker to interact with Google and GitHub accounts. Leave values blank until you create the right account credentials.

To generate a Google refresh token, temporarily set EXPOSE_REFRESH_TOKEN_IN_CALLBACK=true in Render, open your backend OAuth start route, finish the Google login flow, copy the refresh token from the callback response, save it in Render, and then set EXPOSE_REFRESH_TOKEN_IN_CALLBACK=false again.

Business tool integrations

These blocks support business workflows like ads, analytics, growth, commerce, payments, and content operations. Add only the services you plan to use.

Voice and phone integrations

Use these when you want the AI worker to support phone calls, voice behavior, Twilio logs, or OpenAI Realtime SIP settings.

Ads, analytics, and growth integrations

Controlled-write defaults are included where useful so the worker starts from a safer operating mode.

Finance and trading integrations

Keep live trading disabled unless you explicitly know what you are doing. Start with paper or low-limit settings.

Final safety checklist

Run this checklist before deploying or sharing anything publicly.

  • Every real secret is stored in Render environment variables, not in GitHub.
  • Public website blocks still show blanks, placeholders, or safe defaults only.
  • EXPOSE_REFRESH_TOKEN_IN_CALLBACK is back to false after Google setup.
  • OAuth redirect URLs match your Render backend domain exactly.
  • Finance, ads, payments, and commerce tools are in the autonomy mode and limit settings you actually want.
